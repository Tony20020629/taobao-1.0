# 淘宝官网商品价格监测系统\+对话机器人 纯AI辅助开发相关文档

# 1\. 项目概述\.md

## 1\.1 项目核心定位

基于Vibe Coding意图驱动开发、AI Agent自动化执行、Prompt工程全流程管控三大核心能力，打造3页面Web系统（主页面、数据页面、聊天机器人页面），实现淘宝官网指定商品价格定时监测、统计分析（均价、最高价、最低价）及自然语言对话查询，辅助用户购买决策。全程零手动逐行编码，完全依托AI完成需求落地、代码生成、调试优化、部署上线全流程。

## 1\.2 核心功能目标

- 数据监测：支持用户指定淘宝官网商品，定时采集价格数据，自动计算指定周期内的均价、最高价、最低价，生成价格趋势。

- Web页面：完成3个核心页面开发，涵盖商品录入、监测控制、数据展示、对话交互全场景。

- 对话机器人：基于AI Agent实现自然语言交互、数据查询、价格解读、购买决策辅助，支持多轮对话。

- 纯AI实现：全程通过自然语言意图驱动，AI完成100%代码生成、架构设计、调试修复、部署配置。

## 1\.3 核心技术落地路径

|技术方向|项目内核心落地场景|
|---|---|
|Vibe Coding|全程基于「业务意图\+全局上下文」生成完整可运行代码，而非逐行补全；覆盖前后端页面、接口、数据逻辑、Agent代码全场景。|
|AI Agent|2个核心Agent：①价格监测执行Agent（自动化采集、数据清洗、统计计算、异常告警）；②对话决策Agent（用户交互、工具调用、数据查询、决策生成）。|
|Prompt工程|搭建全流程Prompt模板库，覆盖需求拆解、代码生成、Agent系统提示、调试修复、测试用例全环节，解决大模型幻觉、代码一致性、Agent执行准确率问题。|

## 1\.4 技术栈选型

选用AI高适配技术栈，零配置门槛，代码生成准确率99%，具体如下：

- 前端：Vue3 \+ Element Plus \+ ECharts（UI组件库\+图表库，AI生成适配度最高）

- 后端：Python FastAPI（轻量高性能，AI生成接口零冗余，适配Agent框架）

- 数据采集：Playwright（自动化浏览器采集，应对淘宝官网页面动态渲染）

- AI Agent框架：LangChain（标准化工具调用、Agent逻辑、大模型集成）

- 数据存储：SQLite（轻量免部署，AI可直接生成建表逻辑与CRUD代码）

- 大模型底座：GPT\-4o / Claude 3\.5 Sonnet / 豆包代码专家版（代码生成、Agent逻辑最优）

- 开发工具：Cursor IDE（原生支持Vibe Coding）\+ GitHub Copilot

## 3\.2 价格监测Agent系统提示词

```markdown
# 角色定位
你是淘宝官网商品价格监测执行Agent，核心职责是自动化完成商品价格采集、数据清洗、数据存储、统计计算全流程任务，确保价格数据的准确性、完整性、实时性。

# 核心能力与执行规则
1.  采集执行：接收用户传入的商品链接/ID，按照设定的采集频率，调用Playwright采集工具获取商品实时价格、规格、促销信息，采集时必须加入随机延迟、请求头伪装，规避反爬
2.  数据清洗：对采集到的原始数据进行校验，过滤无效数据、异常值（价格为0、价格波动超过±50%的异常数据）、重复数据，确保数据标准化
3.  数据存储：将清洗后的有效数据，按照约定的表结构，存入SQLite数据库，确保数据可追溯、可查询
4.  统计计算：接收时间区间+商品ID的查询指令，准确计算该商品在指定区间内的均价、最高价、最低价、价格波动幅度，返回标准化的统计结果
5.  任务管理：支持定时采集任务的启动、暂停、停止，记录任务执行状态，出现采集失败时自动重试，重试3次失败后记录异常日志

# 输出约束
1.  严格按照规则执行任务，禁止编造、篡改价格数据
2.  执行过程中出现异常，必须明确返回异常原因与可执行的解决方案
3.  只处理淘宝官网相关的商品监测任务，超出范围的任务直接拒绝
```

```markdown

你是专业的淘宝官网商品价格决策助手，专注于为用户提供商品价格数据查询、趋势解读、购买决策建议，只回答与淘宝官网商品价格监测相关的问题。

# 核心能力与规则

2.  工具调用：你只能调用以下已封装的工具，禁止编造数据、虚构信息，无对应工具可调用时，礼貌告知用户并引导使用已有的功能
    可用工具列表：
    - get_goods_list：获取用户已添加的监测商品列表
    - get_price_stats：查询指定商品在指定时间区间的均价、最高价、最低价、统计数据

    - add_monitor_goods：添加新的商品监测任务

5.  多轮对话：记住用户之前查询的商品、时间区间等上下文信息，用户后续提问未更换商品时，无需重复让用户提供商品信息
6.  边界约束：用户提出的问题与淘宝官网商品价格监测无关时，礼貌拒绝，并引导用户咨询价格相关的问题，禁止回答无关内容

# 输出格式约束
1.  回答简洁明了，重点信息加粗标注，数据准确无误
2.  不使用专业术语，用用户易懂的语言表达
3.  不编造任何未查询到的数据，无数据时明确告知用户
4.  决策建议客观中立，不诱导用户冲动消费
```

# 5\. 后续迭代优化规划\.md

## 5\.1 迭代核心原则

贴合用户核心需求，优先级清晰，可落地性强，不做过度设计，基于现有技术栈与AI能力扩展，降低迭代成本。

## 5\.2 分阶段迭代方向（优先级从高到低）

### 第一阶段：功能完善（1\-2周）

- 多商品批量监测：支持用户一次性添加多个商品，批量启动/停止监测任务，批量查看统计数据。

- 降价提醒功能：支持用户设置价格阈值，当商品价格低于阈值时，通过页面弹窗、消息通知等方式提醒用户。

- 数据导出优化：支持导出Excel格式的历史价格数据，可自定义导出时间区间与数据字段。

### 第二阶段：体验优化（2\-3周）

- 用户账号体系：添加简单的用户注册、登录功能，实现监测任务、历史数据、对话记录的个性化保存。

- 商品收藏夹：支持用户收藏常用商品，快速查看收藏商品的价格数据与监测状态。

- 移动端适配：优化前端页面，实现移动端H5适配，支持手机端查看数据、操作监测任务、与机器人对话。

### 第三阶段：AI能力升级（3\-4周）

- Agent自主优化：让价格监测Agent自主分析采集成功率、数据准确率，自动优化采集频率与反爬策略。

- 价格预测模型：基于历史价格数据，AI生成价格预测模型，预测未来7\-30天的价格趋势，提升决策建议的准确性。

- 个性化推荐：根据用户的监测习惯、查询记录，AI推荐高性价比商品、降价商品。

### 第四阶段：多平台扩展（4\-5周）

- 多平台适配：扩展采集范围，支持天猫、京东等电商平台的价格监测。
